Modal Customization: Ant Design vs ShadCN

Comparativa práctica de personalización de modales

A

Ant Design Modal

Implementación básica:

<Modal
  title="Título del Modal"
  open={isModalOpen}
  onOk={handleOk}
  onCancel={handleCancel}
  okText="Confirmar"
  cancelText="Cancelar"
>
  <p>Contenido del modal</p>
</Modal>

Personalización limitada:

<Modal
  className="custom-modal"
  bodyStyle={{
    padding: '24px',
    background: '#f5f5f5'
  }}
  maskStyle={{
    backgroundColor: 'rgba(0,0,0,0.8)'
  }}
>
Limitaciones:
  • CSS específico requerido
  • Estructura HTML fija
  • Estilos globales conflictivos
  • Difícil personalización profunda
S

ShadCN Dialog

Implementación composable:

<Dialog open={isOpen} onOpenChange={setIsOpen}>
  <DialogTrigger asChild>
    <Button>Abrir Modal</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Título</DialogTitle>
    </DialogHeader>
    <p>Contenido personalizable</p>
  </DialogContent>
</Dialog>

Personalización total:

<DialogContent className="sm:max-w-md">
  <div className="flex items-center space-x-2">
    <div className="grid flex-1 gap-2">
      <Label htmlFor="link">Link</Label>
      <Input id="link" defaultValue="..." />
    </div>
    <Button size="sm" className="px-3">
      <Copy className="h-4 w-4" />
    </Button>
  </div>
</DialogContent>
Ventajas:
  • Composición flexible
  • Tailwind CSS nativo
  • Accesibilidad incluida
  • Control total del markup

Ejemplo: Modal de Confirmación Personalizado

Con Ant Design:

• Requiere CSS custom para colores

• bodyStyle para padding personalizado

• className global para estilos

• Conflictos con tema existente

Con ShadCN:

• Clases Tailwind directas

• Composición de componentes

• Variantes predefinidas

• Consistencia automática

Resultado: 70% menos código CSS personalizado
